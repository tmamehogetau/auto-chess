# Shared Board 技術検証チェックリスト (Day2)

## 1. 目的

共有ボード機能における以下の技術検証を実施:

- **競合検証**: 複数ユーザー同時操作時のロック機構とエラーハンドリング
- **遅延検証**: ネットワーク遅延環境での挙動とカーソル同期の遅延許容性

## 2. 事前準備

### 2.1 サーバ起動

```bash
# ターミナル1: Colyseus サーバ起動
npx tsx server.ts

# ターミナル2: クライアント配信サーバ起動
node scripts/serve-client-check.mjs
```

### 2.2 ブラウザ準備

- **推奨ブラウザ**: Chrome / Firefox
- **タブ数**: 3〜4タブ（各タブを別プレイヤーとして扱う）

### 2.3 アクセスURL

```
http://localhost:8080/src/client/shared-board-check.html?endpoint=ws://localhost:2567&roomName=shared_board&autoconnect=1
```

> **注意**: 各タブで上記URLを開き、自動接続で異なるプレイヤーIDが付与されることを確認

## 3. 競合シナリオ

### 3.1 同時配置競合

**手順**:
1. P1, P2 で同じターゲット選択状態にする
2. P1 が配置実行
3. P2 が0.5秒以内に配置実行を試みる

**期待挙動**:
- P2 に `TARGET_LOCKED` エラーが返る
- P1 の配置が正常に完了
- ターゲットの状態が P1 の配置結果で確定

### 3.2 同時選択競合

**手順**:
1. P1, P2 で連続して同じターゲットをクリック（1秒以内）
2. 両者のカーソル位置を確認

**期待挙動**:
- 最初の選択が有効（アクティブプレイヤー権限）
- 遅い側は選択できないか、競合検出メッセージ表示

### 3.3 リセット競合

**手順**:
1. いずれかのプレイヤーがターゲット選択中
2. 別のプレイヤーがリセットボタン押下

**期待挙動**:
- 全プレイヤーの選択状態がクリア
- ステートが初期状態に戻る

## 4. 遅延シナリオ

### 4.1 Slow 3G 検証

**手順**:
1. Chrome DevTools (F12) → Network → Throttling → "Slow 3G" を選択
2. P1 で操作実行
3. 応答時間とカーソル同期遅延を計測

**期待挙動**:
- 要求が正常にサーバ到達
- `TARGET_LOCKED` などのエラーが正しく返却
- カーソル同期の見た目遅延が許容範囲内（目安: 200ms以内）

### 4.2 Custom 遅延検証

**手順**:
1. Network Throttling → "Custom" を選択
2. Download: 500kbps, Upload: 200kbps, Latency: 1000ms を設定
3. 複数プレイヤーで操作実行

**期待挙動**:
- 高遅延環境でもロック機構が正常動作
- 再試行ロジックが機能（lockDuration=1秒経過後、空きセル条件で成功）

## 5. 観戦者シナリオ

### 5.1 観戦者操作拒否

**手順**:
1. 4つ目のタブを開き観戦者状態（P4）にする
2. P4 でターゲット選択・配置を試みる

**期待挙動**:
- 操作が拒否され `NOT_ACTIVE_PLAYER` エラーが返る
- カーソル移動のみが許可（観戦権限）
- 盤面の状態は観察可能

## 6. 合否基準

| 判定項目 | PASS | FAIL |
|----------|------|------|
| TARGET_LOCKED エラーが適切に返る | 競合時に正しいエラーメッセージ | エラーなし、または誤ったエラー |
| lockDuration(1秒)経過後、空きセルへ再試行成功 | 空きセルへの再試行で操作成功 | 空きセルへ再試行しても失敗 |
| spectator の操作が NOT_ACTIVE_PLAYER | 観戦者操作でエラー | 操作が許可される |
| カーソル同期の見た目遅延 | 200ms以内 | 200ms超、または同期不全 |
| 複数プレイヤー同時操作時の整合性 | 最終状態が一貫 | 不整合、または競合不検出 |
| リセット操作の即時性 | 全プレイヤーで即時クリア | 一部プレイヤーに遅延 |
